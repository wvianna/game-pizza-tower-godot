GZ-001 -> GZ-002 -> GZ-003 -> GZ-004 -> GZ-005

Regras:
- Uma issue por vez.
- Nao avancar se o criterio de aceite da issue atual nao foi atendido.
- Ao final de cada issue, gerar relatorio em docs/relatorios/GZ-XXX.md.

Formato do relatorio:
- Issue
- Objetivo
- Arquivos alterados
- Validacao de aceite
- Status final (Done/Blocked)
- Proximo passo

Ao concluir a sprint:
- gerar docs/relatorios/sprint-01-resumo.md com:
  - issues concluidas;
  - issues bloqueadas;
  - riscos para Sprint 2;
  - recomendacao de inicio da Sprint 2.
