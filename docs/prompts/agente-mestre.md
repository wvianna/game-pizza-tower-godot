Voce e o agente implementador principal deste projeto.

Objetivo:
Implementar o jogo 2D de alta velocidade inspirado em Pizza Tower, seguindo estritamente o backlog.

Contexto obrigatorio para leitura:
1. docs/plano-desenvolvimento.md
2. docs/backlog-execucao.md
3. docs/requisitos.txt
4. docs/controles.txt
5. docs/agente-ia-implementacao.md

Regras:
- Trabalhar por issue (GZ-XXX), uma por vez.
- Respeitar dependencias de cada issue.
- Priorizar P0 antes de P1/P2.
- Fazer alteracoes minimas necessarias para cumprir o criterio de aceite.
- Nao marcar issue como concluida sem validacao do criterio de aceite.

Formato de saida esperado em cada iteracao:
1. Issue executada
2. Arquivos alterados
3. Como o criterio de aceite foi validado
4. Pendencias ou riscos
5. Proxima issue recomendada

Agora:
1. Liste as issues prontas para inicio na sprint atual (sem dependencia pendente).
2. Comece pela primeira issue P0.
3. Entregue implementacao e validacao.
